# OIC NetRadar — Phase 1

FastAPI active monitoring service with static-IP inventory, concurrent ICMP/TCP diagnostics, append-only status history, polling debouncing, and WebSocket dashboard events.

## Run locally

```powershell
cd backend
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
uvicorn app.main:app --reload
```

Open `frontend/index.html` in a local static server (for example VS Code Live Server). Add devices from the dashboard and use **Poll now**. For production, set `DATABASE_URL` to PostgreSQL with `asyncpg`; run migrations before deployment (the included development schema is created automatically).

Before rollout, obtain and validate the VLAN-to-VLAN reachability matrix: ICMP, SNMP UDP 161/162 and syslog UDP 514 must be routed/allowed from every monitored VLAN.

## Phase 2 notifications

Create a `NETWORK_AGENT` using `POST /api/agents` in `/docs`, with an email/phone and a matching `branch_id` (or no branch for the global fallback). A new HIGH criticality DOWN incident opens a ticket, emails the Level 1 agent, and attempts SMS delivery three times. An OPEN ticket automatically escalates to a `SUPERVISOR` or `MANAGER` after `LEVEL_2_ESCALATION_MINUTES`.

Set SMTP and SMS gateway values in `.env` before sending real notifications. Until configured, all attempts are safely recorded as `SKIPPED_NOT_CONFIGURED` in `notification_log` rather than being sent.

## Phase 3 passive monitoring and dependencies

The backend receives Syslog on UDP `5514` and raw SNMP traps on UDP `1162` by default (unprivileged development ports). Set `SYSLOG_PORT=514` and `SNMP_TRAP_PORT=162` only when the production service account/network policy allows those ports. Set `PASSIVE_SOURCE_ALLOWLIST` to a comma-separated list of switch/server addresses before production; an empty value is development-only and permits all sources.

Assign each endpoint's `parent_switch_id` when adding inventory. If its parent switch is DOWN, endpoint-level polling is recorded as `DEPENDENCY_OUTAGE` but no child ticket/alert is created. A failed switch/route event reports its number of active downstream devices.

## Phase 4 production controls

Set `AUTH_REQUIRED=true`, replace `JWT_SECRET`, `ADMIN_PASSWORD`, and `SERVICE_API_KEY`, and terminate TLS at an approved reverse proxy/load balancer. Obtain a token from `POST /api/auth/token` and send it as `Authorization: Bearer <token>`; service-to-service calls may use `X-API-Key`. The independent `watchdog.py` must run as a separate Windows Scheduled Task or service and checks `/health` every minute.

Set `IT_SUPPORT_WEBHOOK_URL` to create tickets in the existing system. The webhook receives an `Auto-generated by NetRadar` tagged incident. Download `GET /api/reports/monthly.pdf` for the baseline management PDF report.

When authentication is enabled, the dashboard presents a login page. Use the `ADMIN_USERNAME` and `ADMIN_PASSWORD` configured in `.env`; its short-lived access token is stored only for the browser session.

## Production deployment and SNMP profile policy

Apply [001_production_schema.sql](migrations/001_production_schema.sql) through the approved PostgreSQL migration process. Configure each VLAN with `POST /api/vlan-profiles`; `snmp_credential_ref` must be a reference to a secrets-manager entry, never an SNMP community string or password. The provided Caddy configuration is a TLS reverse-proxy starting point; replace `netradar.example.internal` with the approved DNS name and organization-managed certificate/TLS policy.
